# Контейнерный класс

## Условие

- Лаба на проектирование и реализацию шаблонного контейнерного класса.
- Перед сдачей кода показываются диаграммы класса и описание интерфейса.
- Очень жестко смотрит на правильность реализации шаблонов, обработку исключений и полноту интерфейса.
- Контейнер должен поддерживать полный набор операций для него свойственных
- Обязательна реализация итераторов для обхода контейнера.
- Необходимо обеспечить механизмы работы с объектами разных типов через шаблонизацию.

## Требования:

- Шаблонный класс (`template`) с параметризацией типов.
- Накладывание ограничений на типы (`concepts`).
- Избыточный интерфейс (методы, операторы).
- Перегрузка операторов ([], (), \==, !=, присваивание и т.д.).
- Обработка исключительных ситуаций (`try-catch`, кастомные exceptions).
- Динамическое управление памятью.
- Правильная работа конструкторов и деструкторов.
- Обязательно добавить концепт для контейнерного класса и проверять ваш класс на его соответствие с помощью `static_assert`
- Обязательно проверять итератор на соответствие его типу с помощью `static_assert`

## Варианты контейнеров:

- **Вектор**
- **Список**
- **Матрица** (самый сложный)
- **Множество**
- **Дерево** - на 3 дополнительных балла (в нашем году убрали)

Способ определения варианта: **(Номер по журналу - 1) % 4 + 1**
