# Каркасный 3D Viewer - ООП

## Условие

Реализовать приложение, в котором можно будет добавить трёхмерный объект (может быть несколько, должна быть возможность переключения между объектами). Объект можно будет перенести, повернуть и масштабировать. Также должна быть реализована возможность добавления камеры и её перемещения по экрану (камер неограниченное количество, должна быть реализована возможность переключения между камерами).

Необходимо переработать первую лабораторную работу (3D Viewer), используя объектно-ориентированное программирование.

## Требования

- Построить модель, которую можно было бы модифицировать, не изменяя написанный код.
- Проектировать таким образом, чтобы продумывать дальнейшую модификацию - выделять абстракции.
- Появится много посредников, поскольку потребуется разделение обязанностей - один объект может выполнять несколько ролей.
- Сразу проектировать таким образом, чтобы в дальнейшем использовать в курсаче
- Реализуется с использованием паттернов проектирования и ООП.
- Перед выполнением лабораторной работы нужно продемонстрировать хотя бы раз схему, в которой будут указаны все связи между классами внутри кода программы.

Работа лабораторной состоит из двух задач:

1. **Сделать диаграмму** для написания программы (нужно утвердить её, потом приступать к написанию кода)
2. **Написать код программы** по диаграмме
