# AGENTS.md

## Контекст лабораторной

Цель: реализовать программу для работы со сценой из трехмерных каркасных объектов с использованием паттернов и шаблонов проектирования.

Возможности:
- вращение, перемещение и масштабирование сцены/объектов;
- сцена содержит 3D каркасные объекты;
- до начала кодирования должна быть UML-диаграмма классов всей системы;
- на сцене можно создавать или загружать композиты (объект из других объектов, включая камеры).
- камера должна уметь поворачиваться вокруг своей точки.
- удаление невидимых ребер;
- множественные источники данных (txt/bin/csv/obj и т.д.);
- множественные внутренние представления модели (матрица, списки)
- паттерн Composite 
- множество камер 
- история операций и undo через паттерн Memento/Caretaker

## Анализ эталона (AAETALON)

## Глобальное удаление невидимых рёбер (2026-04-22)

Проблема:
- скрытие рёбер считалось отдельно для каждого объекта, поэтому взаимное перекрытие объектов не учитывалось.

Исправление:
- `DrawVisitor` переведен на двухфазную схему:
  - при обходе сцены для каждого каркасного объекта только накапливаются projected-вершины и ребра в общие буферы;
  - после завершения обхода (в деструкторе visitor) `RemovingInvisibleLinesStrategy` вызывается один раз для всей совокупности рёбер/вершин сцены;
  - на экран выводятся уже глобально отфильтрованные видимые ребра.

Результат:
- скрытие работает между разными объектами сцены, а не только внутри каждого объекта.

## Диагностика "камера не влияет" (2026-04-22, поздно)

Найдены и исправлены критические ошибки в рендер-пайплайне:
- после проекции координаты были в NDC, но `DefaultCenterStrategy` лишь сдвигал их на центр экрана; из-за отсутствия масштабирования в пиксели объекты выглядели "бесконечно далеко" (почти точка);
- этап конвертации координат приведен к логике эталона: теперь NDC -> screen через масштаб `* (width/2, height/2)` и сдвиг в центр;
- поворот камеры в `DefaultCameraEntityStructure::transform` ранее вращал только позицию/центр, но не ориентацию камеры; теперь:
  - `Move` двигает `viewpoint`,
  - `Rotate` обновляет `quaternion` (через delta-кватернион из матрицы вращения),
  - `Scale` для камеры игнорируется;
- view matrix переведена на корректный `inverse(rotation)` + трансляция через dot-product с позицией камеры;
- FOV камеры приведен к эталонному значению 90 градусов.

Итог:
- камера влияет на сцену как при перемещении, так и при повороте;
- масштаб проекции визуально корректный, модель не "уходит в бесконечность".
